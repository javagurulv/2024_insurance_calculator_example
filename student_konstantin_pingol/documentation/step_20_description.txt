**Задание**
Узнать почему stateless объекты считаются потокобезопасными (thread safe), а stateful объекты не являются
потокобезопасными (not thread safe).

**Краткое описание процесса выполнения**
С помощью интернет источников узнал про различия stateless и stateful объектов.


**Что применил/использовал, чему научился в процессе выполнения задания?**
Используя сведения из учебных материалов узнал, что stateless-объекты не хранят состояние между вызовами, поэтому ни
один поток не может повлиять на результат другого потока вызывая методы stateless объекта, из этого следует, что
stateless объекты потокобезопасны, соответственно, так как stateful-объекты хранят сведения о своем состоянии, то не
 являются потокобезопасными объектами.
