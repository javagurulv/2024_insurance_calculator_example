**Комментарий в свободной форме**
Базовые моменты которые стали понятней:
Gradle - автоматическая система сборки и управления зависимостей проекта.

Можно использовать локально установленный Gradle(через команду gradle в командной строке либо через UI в IDEA),
но можно(и даже лучше) использовать переносной дистрибутив(все, что нужно для запуска мы получаем одним файлом, тем самым обеспечиваем единство версий для проекта).

Как использовать переносной дистрибутив:
это Gradlew(файл с расширением .bat для windows, без этого расширения - для unix систем)
в командной строке пишем ./gradlew и таску, которую хотим выполнить(build, clean, test etc.)

При запуске команды build выполняется файл .jar, он идет в файл с расширением .properties и выполняет сборку проекта с соответствующими настройками в файли .properties
Команда собирает проект(компилирует код, подгружает нужные зависимости и собирает это в единый jar-файл).
